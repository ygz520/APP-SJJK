## 📌 第五部分：计算题核心公式

### 一、挣值管理（EVM）⭐⭐⭐超级重点

**基础指标**：

| 指标 | 英文 | 公式 | 含义 |
|-----|------|-----|-----|
| 计划价值 | PV | - | 截止某日期应完成的工作价值 |
| 挣值 | EV | - | 实际完成工作的价值 |
| 实际成本 | AC | - | 实际花费 |
| 预算价值 | BAC | - | 项目总预算 |

**偏差分析**：

| 指标 | 公式 | 判断标准 |
|-----|------|---------|
| 进度偏差SV | SV = EV - PV | >0 好；<0 差 |
| 成本偏差CV | CV = EV - AC | >0 好；<0 差 |
| 进度绩效指数SPI | SPI = EV/PV | >1 好；<1 差 |
| 成本绩效指数CPI | CPI = EV/AC | >1 好；<1 差 |

**预测分析**：

| 指标 | 公式 | 适用条件 |
|-----|------|---------|
| ETC（完工尚需估算） | (BAC-EV)/CPI | 当前效率持续 |
| ETC | EAC - AC | 特殊情况 |
| EAC（完工估算） | BAC/CPI | 典型偏差 |
| EAC | AC + ETC | 非典型偏差 |
| EAC | BAC | 计划不变 |
| EAC | AC + (BAC-EV)/(SPI×CPI) | 综合考虑 |
| VAC（完工偏差） | VAC = BAC - EAC | - |

**关键效率指标**：
- TCPI（完工尚需绩效指数）= (BAC-EV)/(BAC-AC)

### 二、关键路径法（CPM）

**关键路径计算**：
1. ES（最早开始）= 前序活动EF的最大值
2. EF（最早完成）= ES + 活动时间
3. LS（最晚开始）= LF - 活动时间
4. LF（最晚完成）= 后序活动LS的最小值
5. **总时差 = LS - ES = LF - EF**
6. **自由时差 = 后继活动ES - 本活动EF**

**关键路径特征**：
- 总时差为0的活动
- 可能有多条关键路径
- 项目工期 = 关键路径长度

### 三、决策树分析（EMV）

**步骤**：
1. 绘制决策树（□决策点 ○机会点）
2. 标注各分支概率和收益
3. 从右向左计算期望值
4. 选择期望值最大的方案

**公式**：EMV = Σ(概率 × 收益)

### 四、沟通渠道数

**公式**：n × (n-1) / 2
- n = 沟通相关方数量（含项目经理）

### 五、投资回报（ROI）

| 指标 | 公式 |
|-----|------|
| 投资回报率 | ROI = (年均收益 - 年均成本) / 投资额 × 100% |
| 投资回收期 | 累计净收益等于初始投资的时间 |
| 净现值（NPV） | Σ(收益/(1+r)^t) - 初始投资 |
| 效益成本比（BCR） | 总收益 / 总成本 |

### 二、易混淆概念对比

| 概念对比 | 区别 |
|---------|------|
| 项目章程 vs 项目管理计划 | 章程序授权、发起人制定；计划是执行蓝图、PM制定 |
| 范围蔓延 vs 范围潜变 | 蔓延是无审批增加；潜变是客户无意识增加 |
| 关键路径 vs 关键链 | 关键路径是网络分析；关键链考虑资源约束 |
| 应急储备 vs 管理储备 | 应急储备应对已知风险（成本基准）；管理储备应对未知风险（不纳入基准） |
| 确认范围 vs 控制质量 | 确认范围是验收可交付成果；控制质量是检查技术正确性 |
| 合同收尾 vs 行政收尾 | 合同收尾针对采购合同；行政收尾针对整个项目 |
| 团队管理 vs 资源管理 | 团队管理针对项目团队；资源管理针对所有资源 |

### 一、找错题答题框架

**问题模式**：请指出XXX项目在XXX方面存在的问题

**答题结构**：
```
1. 问题：XXX做法不正确
   依据：XXX理论/原则/标准要求...
   分析：该项目实际情况是...导致的问题
   建议：应该...做

2. 问题：XXX做法不正确
   依据：...
   分析：...
   建议：...
```

### 二、计算题答题要点

1. **先列公式**，再代入数据
2. **计算过程清晰**，单位一致
3. **判断标准要明确**：SPI>1好、CPI>1好
4. **给出结论**：进度超前/落后、成本节约/超支
5. **提出建议**：需要采取什么措施

### 三、论文写作结构（万金油模板）

**摘要（300字）**：
- 项目背景（100字）
- 项目规模、周期、投资
- 本人角色
- 主要成果

**正文（2000字）**：
- 开头（200字）：项目背景、为什么选择该知识领域
- 理论论述（400字）：该知识领域的基本概念、过程
- 实践应用（1200字）：实际工作中的应用（结合题目要求）
  - 计划阶段：做了什么、产出什么
  - 执行阶段：遇到什么问题、如何解决
  - 监控阶段：发现了什么偏差、采取了什么措施
- 收尾总结（200字）：项目成果、经验教训、该领域的理解

### 二、临场考试技巧

**上午选择题（75题/150分钟）**：
- 平均1题2分钟
- 遇到不会的先标记，最后回头看
- 计算题确保不失分
- 法律法规题目注意细节

**下午案例分析（3题/90分钟）**：
- 先看问题再看材料
- 计算题必须写出公式和步骤
- 找错题要写出依据
- 字迹清晰、分点作答

**下午论文（1题/120分钟）**：
- 摘要控制在300字内
- 开头结尾要完整
- 内容要与题目要求匹配
- 准备2-3个不同主题的项目素材
